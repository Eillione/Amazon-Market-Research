# normal 亚马逊市场调研

## 触发与目标

当用户说"请帮我调研 xxx 市场""调研 xxx 亚马逊市场""分析 xxx 市场""帮我做 xxx 市场调研"时启动本 Skill。`xxx` 应理解为 Amazon 产品英文关键词，默认站点为 Amazon US，除非用户指定其他站点。

参数提示：`[Amazon product English keyword] [site=US]`。

目标是生成一份决策级 HTML 报告，结构参考当前项目中的 Magnetic Drawing Board 报告模板，最终覆盖从市场细分到产品定义与产品策略的完整链路。进入"最终产品定义和产品策略"阶段时，必须叠加使用 `product-strategy` Skill 的产品策略框架。

详细报告结构和表格模板见 [report-workflow.md](references/report-workflow.md)。启动后必须读取该文件。

## 自动流转规则

本 Skill 采用自动流转模式，无需人工干预即可完成完整调研流程。

1. 先完成"关键词市场总览、细分市场分类、细分市场增长率与趋势动能、进入优先级排序"。
2. 根据进入优先级排序结果，自动选取排名最高的细分市场继续深研，直至该细分市场的亚马逊站点体量、关键词、核心竞品信息都被调研分析完成。
3. 继续完成竞品深拆、用户画像、Kano、VOC、Reddit/社媒反馈、新品卖点排序。
4. 自动基于已分析数据生成产品规划初稿，系统根据市场分析和VOC结果自动推导1-3个SKU方案，并在报告末尾自动生成可填写的"产品规划表单"，优先复用 `assets/product-plan-form.html` 的结构和交互。表单必须支持 1-3 个 SKU 横向填写，并包含这些字段：SKU/型号、产品定价（USD）、核心卖点、次要卖点、产品配置&性能（选填）、销售渠道、预估毛利率、上市时间。表单应支持本地自动保存、复制已填写内容、清空表单。用户可在报告交付后修改表单内容并重新触发更新。
5. 继续输出最终产品定义、SKU 策略、定价渠道、利润成本核算、上市节奏和完整 HTML 报告。该阶段必须包含 `SKU 规划与产品定义` 板块，并严格使用 `report-workflow.md` 中的 `5.1C SKU 规划与产品定义固定格式`：三列表格 `产品规划 / 基础款 / 升级款`，固定行顺序，核心卖点使用 `sell-grid` / `sell-item` 卡片。无论用户提供 1 个、2 个、3 个或更多 SKU，都必须映射进该固定格式，不得改成普通 SKU 矩阵表或散文段落。若用户只填写"预估毛利率"而未提供采购价、FBA、头程等详细成本，利润成本核算先按用户毛利率做策略级估算，并标注详细费用待验证。该阶段必须显式按 `product-strategy` 框架补齐策略诊断、目标用户、竞争逻辑、产品屋、产品矩阵、生命周期、定价传播、30/60/90 天动作和一页式策略全景图。
6. 最终完整报告必须包含"合规风险分析"章节。不得只写泛化风险，必须固定拆成 `1. 侵权风险分析`、`2. 安全风险分析`、`3. 其他风险`。侵权风险必须调用或引用 USPTO / EUIPO 官方数据源，覆盖核心关键词商标侵权风险和产品专利/外观设计风险；安全风险必须调用或引用相关法规文件数据源，结合产品形态、目标人群、使用场景、材料结构、平台渠道和国家/地区要求，输出风险等级、触发原因、建议动作和上市前证据清单。
7. 最终交付前必须自检：HTML 中必须存在标题 `SKU 规划与产品定义`，且其主表头必须是 `产品规划 / 基础款 / 升级款`；必须包含 `SKU / 英文命名`、`一句 Slogan`、`角色`、`核心卖点`、`其他卖点`、`定价`、`主要竞品`、`销售渠道` 这些固定行。若缺失，必须先修正报告再交付。

## 数据与分析原则

- 市场口径优先使用 Amazon 关键词自然搜索结果，而不是只按某个 ASIN 所属类目判断市场。
- 细分市场分类应基于关键词搜索结果前几页自然位产品集合，按产品形态、使用场景、价格带和用户需求聚类。
- 做"进入优先级排序"时，必须把"欧美玩具品牌相关性"纳入评估标准。默认品牌为 JoyCat，除非用户明确指定其他品牌。应先通过 `product_search` 按品牌名检索现有产品，提炼品牌当前人群、场景、价格带、品类能力、合规能力和渠道心智，再评估每个细分市场与现有产品线的协同度。
- 进入优先级不能只按市场容量排序。必须同时输出市场综合分、JoyCat/指定品牌相关性分、品牌适配后分数和排序理由；若缺少品牌现有产品数据，必须标注"待验证"，并基于用户已提供信息做假设说明。
- 重点细分市场确定后，再做竞品深拆、用户画像、Kano、VOC、Reddit/社媒反馈、新品卖点排序。
- "主要竞品调研"的竞品 ASIN 必须嵌入 Amazon 商品页超链接，US 站默认使用 `https://www.amazon.com/dp/{ASIN}`，方便从报告直接跳转核验；`规格参数` 必须使用固定字段：产品规格、产品重量、产品材质、FBA费用、产品评分、评论数量、上架时间。优先用 `product_detail` 补齐；缺失时保留字段名并标注"待验证"。
- 儿童玩具、宠物用品、礼品等存在"购买者 ≠ 使用者"的品类，必须分开写画像。
- 儿童、宠物、食品接触、电子、电池、磁体、医疗/健康/感官/治疗宣称、户外/泳池/睡眠/承重家具等品类，必须主动做合规风险分析，并标注需要外部法规顾问或第三方实验室验证的内容。
- 不要凭空编造硬数据。无法采集或不确定的数据标注为"待验证"或"假设"。
- 输出必须落到产品经理可执行的决策：进入哪个细分、为什么、做什么 SKU、用什么卖点、如何定价、成本是否成立、怎么上线验证。

## 输出文件

默认输出到：

```text
reports/dtc/{keyword_slug}_amazon_market_research_{SITE}_{YYYYMMDD}.html
```
同步输出pdf文件到：
```text
reports/dtc/{keyword_slug}_amazon_market_research_{SITE}_{YYYYMMDD}.pdf
```
如用户要求同步飞书，再按现有 lark/whiteboard 工作流另行生成云文档或画板版。
